1. PROJECT OVERVIEW
-------------------
MYLOC.DZ is a car rental website for the Algerian market. It allows users to:
- Browse the vehicle fleet (citadines, SUVs, berlines)
- View car details, prices, and specifications
- Access a client dashboard to manage reservations, profile, payments, and settings
- Access an admin dashboard to manage clients and vehicles
- Navigate through a modern, responsive, animated landing page

The project consists of:
- A static Next.js frontend deployed to GitHub Pages.
- A separate PHP + MySQL REST API backend in `myloc-backend/` that provides authentication,
  car management, and reservation handling.

5. COMPONENTS
-------------
Landing Page Sections:
- Navbar: Fixed top navigation with scroll links, Admin link, and Log in button.
- Hero: Main banner/intro section.
- Fleet: Auto-scrolling horizontal carousel of car cards.
- Features: Selling points / advantages section.
- About: Company information.
- Contact: Contact form/information.
- Footer: Site footer.

- Displays a vehicle with category, name, price, transmission, seats, and year.

6. STATE MANAGEMENT
-------------------
The app uses React Context for simple tab navigation in the client and admin dashboards:
- ClientContext: stores activeTab ("accueil" | "profil" | "reservations" | "reserver" | "paiements" | "parametres")
- AdminContext: stores activeTab ("dashboard" | "clients" | "cars")

Backend integration is ready but the frontend still uses mock data. The Next.js components need to be wired to the PHP API endpoints to use real data.

8. BUILD AND DEPLOYMENT
-----------------------
Build command:
  npm run build

Next.js config (next.config.ts):
  output: "export"       # Static HTML export
  distDir: "dist"        # Output folder
  assetPrefix: "."      # Relative paths for GitHub Pages subfolder
  images: { unoptimized: true }

GitHub Actions workflow (.github/workflows/deploy.yml):
- Triggers on push to main or manual dispatch.
- Sets up Node 20.
- Runs npm install --no-audit --no-fund.
- Runs npm run build.
- Uploads ./dist as a Pages artifact.
- Deploys to GitHub Pages.

The conflicting nextjs.yml starter workflow was removed because it expected output in ./out instead of ./dist.
