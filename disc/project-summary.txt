1. PROJECT OVERVIEW
-------------------
MYLOC.DZ is a car rental website for the Algerian market. It allows users to:
- Browse the vehicle fleet (citadines, SUVs, berlines)
- View car details, prices, and specifications
- Access a client dashboard to manage reservations, profile, payments, and settings
- Access an admin dashboard to manage clients and vehicles
- Navigate through a modern, responsive, animated landing page

The project is built as a static Next.js application and deployed to GitHub Pages.

5. COMPONENTS
-------------
Landing Page Sections:
- Navbar: Fixed top navigation with scroll links, Admin link, and Log in button.
- Hero: Main banner/intro section.
- Fleet: Auto-scrolling horizontal carousel of car cards.
- Features: Selling points / advantages section.
- About: Company information.
- Contact: Contact form/information.
- Footer: Site footer.

CarCard:
- Displays a vehicle with category, name, price, transmission, seats, and year.
- Includes a 3D floating car image animation.
- Has mouse-follow tilt effect using Framer Motion.

Client Dashboard:
- ClientLayout: Sidebar + main content wrapper.
- ClientSidebar: Left navigation with tabs: Accueil, Mon profil, Mes réservations, Réserver, Paiements, Paramètres.
- ClientContent: Switches between tab views with animated transitions.
- AccueilView: Stats cards and latest reservations preview.
- ProfilView: Editable profile form.
- ReservationsView: Reservation history.
- ReserverView: Searchable/filterable car catalog.
- PaiementsView: Payment invoices.
- ParametresView: Settings toggles.

Admin Dashboard:
- AdminLayout: Sidebar + main content wrapper.
- AdminSidebar: Left navigation with tabs: Dashboard, Nos clients, Cars.
- AdminContent: Switches between admin tab views.
- DashboardView: Admin stats (clients, vehicles, reservations, revenue) + fleet health.
- ClientsView: Searchable client table with status and actions.
- CarsView: Vehicle cards with status badges and edit/delete actions.


6. STATE MANAGEMENT
-------------------
The app uses React Context for simple tab navigation in the client and admin dashboards:
- ClientContext: stores activeTab ("accueil" | "profil" | "reservations" | "reserver" | "paiements" | "parametres")
- AdminContext: stores activeTab ("dashboard" | "clients" | "cars")

There is no backend or persistent database yet. All data is hardcoded mock data inside components.

8. BUILD AND DEPLOYMENT
-----------------------
Build command:
  npm run build

Next.js config (next.config.ts):
  output: "export"       # Static HTML export
  distDir: "dist"        # Output folder
  basePath: "/myloc"     # GitHub Pages subpath
  images: { unoptimized: true }

GitHub Actions workflow (.github/workflows/deploy.yml):
- Triggers on push to main or manual dispatch.
- Sets up Node 20.
- Runs npm install --no-audit --no-fund.
- Runs npm run build.
- Uploads ./dist as a Pages artifact.
- Deploys to GitHub Pages.

The conflicting nextjs.yml starter workflow was removed because it expected output in ./out instead of ./dist.

10. KNOWN ISSUES
----------------
- The site uses mock/hardcoded data. No real backend or authentication exists.
- The login page does not actually authenticate users; it links to client.html.
- Admin access is not protected by any role-based authentication.
- Custom domain was previously attempted with "myloc-dz" and caused a redirect issue; now the site uses the default GitHub Pages URL.
